Analysing Pico Inference Results and Evaluating the Model¶

At this point, the model has been successfully deployed to the Pico.

The Pico requires the images in a certain format (a float32 array of 4 dimensions). This set of images is built automatically when the scripts/init_datasets.sh script is run.

The host script scripts/infer_on_pico.py sends 1,000 processed images to the Pico, and collects the inference results in a CSV.

This notebook analyses the results of the CSV by calculating the accuracy, and rendering the confusion matrix and classification report. The final Keras model is also evaluated below for comparison purposes.

Helper Functions¶

The functions to render the confusion matrix, line plot, and classification report are defined below.

In [ ]:
def configure_fig_size():
  # Configure the Seaborn figure size.
  sns.set(rc={
    "figure.figsize":(15, 10), "font.size":20,
    "axes.titlesize":40, "axes.labelsize":20,
    "xtick.labelsize":20, "ytick.labelsize":20
  })

def confusion_matrix(true, pred):
  configure_fig_size()
  sns.heatmap(
    metrics.confusion_matrix(true, pred),
    annot=True,
    xticklabels=classes, yticklabels=classes,
    cmap='Blues', fmt='d'
  ).set(title='Confusion Matrix')

def line_plot(true, pred):
  configure_fig_size()
  (
    sns
    .lineplot(x = true, y = pred, palette = 'Blues', linewidth=2.5, legend=False)
    .set(title='True vs. Predicted', xlabel='True', ylabel='Predicted')
  )

def classification_report(true, pred):
  print(metrics.classification_report(
    true, pred, target_names=classes, zero_division=0
  ))

Running Inference on the Pico¶

To run inference on the Pico, first deploy the UF2 image located at artifacts/diode.uf2 to a Pico. To do this, hold the BOOTSEL button on the Pico and connect it to the computer. Then drag and drop the UF2 file to the newly mounted drive (named RPI-RP2). After that, disconnect the Pico, and then run the script. The script tells you to connect the Pico when it starts. The inference automatically starts after the Pico is detected by the script.

Note: Ensure that the script is run in the Conda environment that's already setup. Follow instructions in readme.md to setup the Conda environment.

Results¶

The inference results of the Pico show a slightly higher accuracy of 93% compared to 91.78%. This could be due to the random selection of the subset of 1,000 images from the entire 2,700 image dataset.

It is important to note that the Universal Workflow of Deep Learning described in the Deep Learning with Python book prescribes re-training the final model with the training and validation dataset combined for the number of epochs that leads to the best accuracy. Since the validation dataset is only 10% of the data, and retraining the models takes a long time, this process is not done here. The model is very effective even without doing this process.

Overall, the 93% accuracy reached by the Pico demonstrates that Deep Learning and specifically Image Classification on a microcontroller is suitable and performant.
